How to setup SSH keys

March 17, 2019

SSH (Secure Shell) is an encrypted network protocol that allows to operate over an insecure network in a secure way.
It is commonly used to administer a remote server using a terminal session, establishing a TCP connection on port 22 in a client-server architecture.
However, it can also be used to create an encrypted tunnel to secure any network communication, or transfer files via related SFTP (SSH File Transfer Protocol) or SCP (Secure Copy Protocol) .

SSH uses public-key cryptography to authenticate users.
Public-key cryptography uses a pair of keys: a public key and a private key. The public key should be put on the remote server to which we want to gain access, the private key should be kept secret and known only to the user.
This setting is similar to a door lock: the public key is the door lock and the private key is the door key. The difference is that the same door can have multiple door locks that open it, each one with an associated with a different door key.

To create a SSH key pair (using the widely available ssh-keygen tool) simply type:

ssh-keygen

You should see the following output:

Generating public/private rsa key pair.
Enter file in which to save the key (/user/.ssh/id_rsa):

By default, it saves the key pair under your home directory in a hidden .ssh folder.
Then it will ask for a (optional) passphrase, used to add an additional layer of protection to the private key. Finally, it should write in output

Your identification has been saved in /user/.ssh/id_rsa.
Your public key has been saved in /user/.ssh/id_rsa.pub.

We have successfully generated a new SSH key pair.
Remember to set the correct permissions on the file system:

chmod 700 .ssh
chmod 644 .ssh/id_rsa.pub
chmod 600 .ssh/id_rsa

To authenticate to a remote server we should now copy the generated public key in a specific location on the remote server, which is the authorized_keys file under ~/.ssh folder.
If the file already exists we should append the new public key to the keys already present in the file.

On our client we should now be able to login to the remote server without using a password:

ssh username@remote_host
