Using Python virtualenv

February 21, 2019

Each Python project comes with its own dependencies, usually listed in a requirements.txt file. As the number of projects grows comes the need to isolate each installation, as each one may use a different version of the same library, and it's simply no more feasible to use a system wide installation.

virtualenv (https://virtualenv.pypa.io/en/latest/) is a Python module which allows easy creation of virtual environments right inside the project folder.

To begin, install virtualenv:

pip install virtualenv

Inside your project folder create a virtual environment (usually named venv as a convention):

virtualenv venv

This creates a separate python executable, and separate pip and setuptools you can use to install dependencies in venv.

How to activate the environment depends on the OS used. On Unix based machines (MacOS, Linux), you can use:

source venv/bin/activate

On Windows machines you have to use a .bat or .ps1 script located at venv/Scripts/activate.bat and venv/Scripts/activate.ps1, respectively.

On either OS, to deactivate the environment simpy use deactivate.

On bash, you can check which enviroment you are using with

which python

Once the virtual environment is active you can use pip to install modules, which will be installed inside the venv folder in a standard site-packages directory. For example, you may want to use

pip install -r requirements.txt

to install the dependencies of your project.

If you need to run a project script with a process control system (such as supervisord), be sure to specify the right python executable to use (i.e. venv/bin/python).
