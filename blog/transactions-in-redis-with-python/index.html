Transactions in Redis with Python

September 21, 2019

Redis is a powerful in-memory data structure store, which is frequently used for storing cache.
It supports different forms of data types (strings, lists, maps etc.), has persistence on physical memory in the form of periodical snapshots and has support for transactions.

redis-py is the most popular Redis client for Python (https://github.com/andymccurdy/redis-py) and makes using this data store from an application very simple.

redis-py can be installed with pip:

pip install redis

The Redis class is the actual client we use to connect to the database:

import redis

r = redis.Redis()

If we don't specify the parameters for the class, default values for the connection will be used (host='localhost', port=6379 and db=0).

To set a key-value pair of type string-string, we simply use the set() method:

r.set(key, value)

Suppose to have a scenario in which multiple clients write to the data store. We would like to avoid possible conflicts in the modifications of the value of a key.
The solution to the problem is to use a transaction. In a transaction we can queue multiple commands that will be executed atomically by the server.
More importantly, we can watch for changes on one or more keys, and if a change in the value is detected during the queueing of the commands, the actual execution will fail.

r.set('string1', 'pine')
r.set('string2', 'apple')

p = r.pipeline()
p.watch('string1', 'string2')  # watch for changes on these keys
string1 = p.get('string1')
string2 = p.get('string2')
p.multi()  # starts transactional block of pipeline
new_string1 = string1 + string2
p.set('string1', new_string1)
p.execute()  # ends transactional block of pipeline

If another client changes the value of string1 or string2 during the queueing of the commands inside the transactional block, the execution fails and needs to be retried.

redis-py offers also a convenience function to shorten the above code, which is redis.Redis.transaction:

def transaction_func(pipe: redis.client.Pipeline) -> None:
    new_string1: str = pipe.get('string1') + pipe.get('string2')
    pipe.set('string1', new_string1)

r.transaction(transaction_func, *['string1', 'string2'])

The 1st argument should be a callable with a single parameter of type redis.client.Pipeline, the second argument should be an unpacked list of keys to watch.
