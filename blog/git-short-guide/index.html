VERSION CONTROL

Git Short Guide

January 10, 2019

Git is a distributed version control system for tracking changes in source code during software development, created in 2005 by Linus Torvalds (https://en.wikipedia.org/wiki/Git).

  • Initialize local repository:
git init
  • Add remote named origin (a remote is simply an alias for a remote repository URL):
git remote add origin <url_to_remote>
  • Fetch commits from remote to local repository:
git fetch
  • Pull and merge commits from remote to local repository and working copy:
git pull
  • Check working directory status:
git status
  • View changes to files not yet staged:
git diff
  • Add all changes to staging:
git add *
  • Commit staged files to local repository:
git commit -m "add missing timezone support"
  • Push local commits to remote repository on branch x:
git push <remote_name> x
  • Delete all your local changes and reset to last commit:
git reset --hard
  • List local branches:
git branch
  • Create a new branch and checkout:
git branch awesome_branch
git checkout awesome_branch

In short: git checkout -b awesome_branch

  • Merge branch_y onto branch_x:
git checkout branch_x
git merge branch_y
  • Delete branch_y:
git branch -d branch_y
  • Rename local branch:
git branch -m <new_name>

if on a different branch from the one to rename:

git branch -m <branch_name> <new_name>
  • Rename remote branch:
git push --delete <remote_name> <branch_name>
